25 // The `nvml` namespace acts as a higher-level wrapper on top of the
26 // NVIDIA Management Library (NVML). By providing this higher-level
27 // API, we eliminate the build-time dependency on `libnvidia-ml` in
28 // favor of a runtime dependency. The caller may probe to see if the
29 // library is available, and may take different code paths accordingly.
30 // This is advantageous for deploying a common `libmesos` on machines
31 // both with and without GPUs installed. Only those machines that have
32 // GPUs need to have the `libnvidia-ml` library installed on their
33 // system.
34 //
35 // TODO(klueska): Add unit tests and provide finalization support.
